Mobile Healthcare Application

For my dissertation "Apps to manage multiple healthcare appointments and documents", I started with research into a demographic and focused on chronic patients. Combined with Figma prototyping and the SDLC, I was able to refine the app as shown below with Role-Based Authentication (Patients, Doctors) and SQL CRUD Operations (Create, Read, Update, Delete).

Summit Seeker

Group | React Native, TypeScript, Expo Go | Jan. 2024 - Apr. 2024

During 3rd Year of University, my group was tasked with creating a mobile application for a target audience. The assignment demanded the use of Application Programming Interface calls, therefore, we chose tourists and frequent travellers. As the module was studying the use of Mobile Geographical data, this was a factor in our decision for our target demographic. The key features for navigation and planning when travelling are shown below.

  • Calendar: Date scheduling
  • Geographical Sensor Data: Read and displayed in case of loss of direction.
  • Map: Routing and points of interest.
  • News: Up-to-date city events.
  • Weather: Event planning, appropriate travel packing.
  • Notes: Journalling and lists.

Art Shop

Individual | HTML, CSS, JavaScript, PHP, MySQL | Aug. 2023 - Dec. 2023

As an introduction to Web Applications Development, the project was to create an interactive E-commerce website that sells artwork. This assignment specified that customers can buy, view and remove artworks from their basket while an admin can carry out SQL CRUD operations on the artworks, reflected to customers.

  • Cards: Used to display the artworks, their price and information on a large scale and avoid repetitive/hard code.
  • Role-Based Authentication (Customers, Admin): No customer login but admin can gain access using a secret password.
  • SQL CRUD Operations (Create, Read, Update, Delete)
  • Shopping Basket: Multiple artworks can be purchased or removed from basket.
